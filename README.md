# DefinitelyEngine

A cross-platform C++17 game engine with a shared library architecture, featuring application lifecycle management and structured logging via spdlog.

---

## Project Structure

```
DefinitelyEngine/
├── Engine/                     # Core engine (builds as a shared library)
│   ├── src/
│   │   └── DefinitelyEngine/
│   │       ├── Application.h / .cpp   # Base Application class
│   │       ├── Entry.h                # Main entry point
│   │       ├── Log.h / .cpp           # Logging system
│   │       └── Core.h                 # Platform export macros
│   └── external/
│       └── spdlog/                    # Logging dependency (git submodule)
├── Sandbox/                    # Sample client application
│   └── src/
│       └── Sandbox.cpp
├── premake5.lua                # Build configuration
└── Makefile                    # Root makefile (generated by Premake5)
```

## Prerequisites

### All Platforms

- **C++17-compatible compiler**
  - macOS/Linux: `clang++` or `g++`
  - Windows: MSVC (Visual Studio 2019+) or MinGW
- **[Premake5](https://premake.github.io/download/)** — build system generator
- **GNU Make** (macOS/Linux) or **Visual Studio** (Windows)
- **Git** — for cloning submodules

### macOS

```bash
xcode-select --install    # Installs clang and make
brew install premake      # Install Premake5 via Homebrew
```

### Linux (Debian/Ubuntu)

```bash
sudo apt update
sudo apt install build-essential git
# Download Premake5 binary from https://premake.github.io/download/
```

### Windows

- Install [Visual Studio 2019 or 2022](https://visualstudio.microsoft.com/) with the **Desktop development with C++** workload
- Download the [Premake5 binary](https://premake.github.io/download/) and add it to your `PATH`

## Setup

### 1. Clone the repository with submodules

```bash
git clone --recurse-submodules https://github.com/abhaychandavar/DefinitelyEngine.git
cd DefinitelyEngine
```

If you already cloned without submodules:

```bash
git submodule update --init --recursive
```

This pulls in `spdlog` under `Engine/external/spdlog/`.

## Building

### macOS / Linux

#### Generate build files

```bash
premake5 gmake2
```

#### Build (Debug)

```bash
make
```

#### Build (Release)

```bash
make config=release
```

#### Build a specific project only

```bash
make Engine           # Build only the engine library
make Sandbox          # Build only the sandbox app
```

#### Clean build artifacts

```bash
make clean
```

---

### Windows (Visual Studio)

#### Generate Visual Studio solution

```bash
premake5 vs2022      # or vs2019
```

Open `DefinitelyEngine.sln` in Visual Studio, select **Debug** or **Release**, and build with `Ctrl+Shift+B`.

## Running

After a successful build, binaries are placed in:

```
bin/[Config]-[OS]-x64/
├── Engine/      # Shared library (.dylib / .so / .dll)
└── Sandbox/     # Executable
```

### macOS / Linux

```bash
# Debug build
./bin/Debug-macosx-x64/Sandbox/Sandbox

# Release build
./bin/Release-macosx-x64/Sandbox/Sandbox
```

> **Note (macOS):** If you see a dynamic library load error, ensure the engine `.dylib` is accessible. You can run from the project root or set `DYLD_LIBRARY_PATH`:
>
> ```bash
> export DYLD_LIBRARY_PATH=bin/Debug-macosx-x64/Engine:$DYLD_LIBRARY_PATH
> ./bin/Debug-macosx-x64/Sandbox/Sandbox
> ```

### Windows

```
bin\Debug-windows-x64\Sandbox\Sandbox.exe
```

## Creating a Client Application

To build your own application on top of the engine:

1. Create a class that inherits from `DefinitelyEngine::Application`:

```cpp
#include <DefinitelyEngine.h>

class MyApp : public DefinitelyEngine::Application {
public:
    MyApp() {}
    ~MyApp() {}
};
```

2. Define the factory function that the engine entry point calls:

```cpp
DefinitelyEngine::Application* DefinitelyEngine::CreateApplication() {
    return new MyApp();
}
```

The engine's `Entry.h` provides the `main()` function — do not define your own.

## Logging

The engine exposes two loggers: **Core** (engine internals) and **Client** (your application).

```cpp
// Client-side logging macros
DE_CLIENT_TRACE("message");
DE_CLIENT_INFO("message");
DE_CLIENT_WARN("message");
DE_CLIENT_ERROR("message");
DE_CLIENT_FATAL("message");

// Engine-side (core) logging macros
DE_CORE_TRACE("message");
DE_CORE_INFO("message");
DE_CORE_WARN("message");
DE_CORE_ERROR("message");
DE_CORE_FATAL("message");
```

## Build Configurations

| Configuration | Preprocessor Define | Optimizations | Debug Symbols |
|---------------|---------------------|---------------|---------------|
| Debug         | `DE_DEBUG`          | Off           | On            |
| Release       | `DE_RELEASE`        | On            | Off           |

Platform defines are set automatically: `DE_PLATFORM_WINDOWS`, `DE_PLATFORM_MAC`, or `DE_PLATFORM_LINUX`.
